Instructions

Getting Started

You can create mock data in two ways:

  1. Manual Creation: Add and configure attributes directly in the application
  2. Import File: Load a pre-configured attribute set from a file

Creating Attributes

  1. Enter the attribute name
  2. Select the appropriate data type
  3. Add possible values or configure ranges
  4. Repeat for all needed attributes
  5. Click "Preview" to generate MockData
  6. Export generated data in the format of your choosing

Import/Export Attributes

You can save and load your attribute configurations using files for convenience:

Supported Input File Types

  • JSON - JavaScript Object Notation files (.json)
  • CSV - Comma Separated Values files (.csv)
  • XLSX - Excel Workbook files (.xlsx)

File Format Requirements

CSV Format

Each line should contain: AttributeName, Type, Value1, Value2, ...

Name,Text,John,Jane,Bob
Age,Numeric,25,30,35
Active,Boolean,true,false

XLSX Format

First row: Attribute names
Second row: Types (optional - defaults to Text)
Following rows: Values

JSON Format

JSON is the Default Import/Export File format

[
{
"attrName": "Name",
"type": "Text",
"values": ["John", "Jane", "Bob"]
}
]

Supported Attribute Types

  • Text - Any string value
  • Boolean - true/false values
  • Bit - 0 or 1 values
  • Numeric - Any number value
  • Integer Range - Generates numbers within specified range
  • Float Range - Generates decimal numbers within specified range
  • Date Range - Generates dates within specified range

Generation Options

  • Output Count - Number of mock records to generate
  • Output Name - The name of the generated file
  • SQL Table Name - The name of the SQL Table your want to insert data
  • Enable ID - Adds an auto-incrementing ID field
  • Probability Function:
    • Flat - Equal probability for all values
    • Linear - Higher probability for earlier values

Tips

  • Ensure all required columns have values
  • For range types, provide minimum and maximum values
  • Use appropriate data types for better mock data generation
  • Review the generated preview before final export
